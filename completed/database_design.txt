## 4) Recommended High-Security Target Configuration

Target security level: HIGH

1. Transport Security
- Enable PostgreSQL SSL and enforce encrypted connections.
- Move client to `DB_SSLMODE=verify-full` and configure CA cert path.

2. Access Control Model
- Keep app on non-superuser role.
- Remove schema DDL rights from runtime role.
- Restrict privileges to exact tables and operations needed.

3. Data Protection
- Keep SCRAM (`scram-sha-256`) and enforce strong password policy.
- Hash session tokens before storing.
- Classify sensitive columns and apply encryption strategy where required.

4. Multi-Tenant/User Isolation
- Implement per-user RLS policies and test bypass resistance.

5. Governance and Operations
- Enable stronger DB logging/audit for auth + DDL events.
- Add automated secure backups + restore testing.
- Add secret rotation and incident-response playbook.
